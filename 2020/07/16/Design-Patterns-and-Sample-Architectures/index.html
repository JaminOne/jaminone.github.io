A. About high availability

Firstly and most importantly, secure HA by using the multi-AZ pattern.

  1. AMI
  2. Load balancer
  3. confirm the healthy state
  4. Create master and standby RDS read replica for automatic failover
  5. Use elastic Ip address. Disassociate the Elastic IP address from the original one, and associate with the new one. Easy fallback procedure.
  6. Stateless (ElastiCache, DynamoDB, store in the data store rather than web/app server)
  7. Scaling by schedule or by policy
  8. Use Amazon SQS to do the batch job
  9. Bootstrap
  10. bootstrap instance

D. Case study: COVID19 Outbreak heatmap

Traditional method: Regional governors send data to the FTP server. It’s highly like to cause single point of failure. Aggregating data is delay and slow. AWS Kinesis can generate real time data but the payload size is limited. The global data may exceed the limit. Cloudfrond is compatible with this case, but it doesn’t cache the data. There are lots of tradeoff when you look into this case.
